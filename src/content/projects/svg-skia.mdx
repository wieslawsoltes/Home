SVG is both a document format and a rendering contract. The library keeps those concerns close enough for fidelity, yet separate enough that UI controls, build tools, and editors can share the same interpretation.

<StoryDecision label="Fidelity rule" title="Test the picture, not only the parser.">
  A valid DOM is not proof of a correct image. Svg.Skia uses W3C coverage and resvg comparison lanes to keep rendering behavior visible, because **the final pixels are part of the public API**.
