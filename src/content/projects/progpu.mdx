The useful unit is not a drawing API. It is the **whole route from application intent to a presented frame**, with enough separation that a framework, editor, or experiment can enter at the right level.

<StoryDecision label="Design constraint" title="Own the frame without owning the application.">
  ProGPU keeps the low-level route visible, but it does not require every consumer to become a renderer. **Each package is a replaceable seam**: use the backend alone, compose a retained scene, or connect an established UI framework.
</StoryDecision>
